## Phase 1 — Build the screens (start here)

Implement in React Native, matching the prototype visually:

### Workout Execution screen (`WorkoutSession.tsx`)
- Running total workout timer (starts on mount, counts up)
- Scrollable list of exercises in the session
- Each exercise shows its name, set rows, and a "+ Add Set" button
- Tap a set row to open the set logging flow
- Rest timer countdown between sets (configurable seconds; audio/haptic on zero — use `expo-haptics` and `expo-av`)
- "End Workout" button that navigates to the summary screen

### Set Logging (`SetRow.tsx` or inline)
- Reps input and weight input (numeric keyboard)
- Show the most recent logged values for this exercise inline as greyed ghost text or a "last time" label
- "Mark done" / checkmark action
- Sets can be added, edited, or removed while the session is active

### Post-Workout Summary (`WorkoutSummary.tsx`)
- Total sets completed
- Total reps
- Total volume lifted (sum of reps × weight_kg across all sets)
- Total workout duration (formatted as mm:ss or hh:mm:ss)
- "Save Workout" button (Phase 1: saves to local state; Phase 2: saves to Supabase)

Use data shaped exactly like `volt-data.js` for local mock state in Phase 1.

## Phase 2 — Wire up Supabase (only after api-contract.md is ready)

1. Read `Team/api-contract.md` fully before writing any client code.
2. Create `src/lib/supabase.ts` following the setup in the contract exactly.
3. Environment variables go in `.env` at the project root — **never hardcoded**:
   ```
   EXPO_PUBLIC_SUPABASE_URL=...
   EXPO_PUBLIC_SUPABASE_ANON_KEY=...
   ```
   Access via `process.env.EXPO_PUBLIC_SUPABASE_URL` in `app.config.js` and expose to the app through Expo's extra config.
4. Implement auth screens: sign-in, sign-up using Supabase email + password.
5. Replace mock data hooks with real Supabase queries per the contract.
6. Persist an in-progress workout session to Supabase on every set completion, so the app can recover if backgrounded or killed.
7. Query previous performance: fetch the most recent `workout_sets` row for each exercise to show as the "last time" reference.
